üìö What Are Functions?

A function is a reusable block of code that performs a specific task. Instead of writing the same code over and over, you can define a function once and call it whenever you need it.

Think of it like a recipe: The recipe has a name ("Make Sandwich"), ingredients (parameters), and produces a result (return value). You can use the recipe many times without rewriting it.

Why Use Functions?

  • Reusability: Write code once, use it many times
  • Organization: Break complex problems into smaller pieces
  • Readability: Give meaningful names to blocks of code
  • Testing: Test individual functions independently
  • Maintenance: Fix bugs in one place instead of many

üî® Creating Functions

Basic Function Syntax

Use the def keyword to define a function:

def greet():
    print("Hello, World!")
    print("Welcome to Python!")

# Call the function
greet()

Function Naming Conventions

Good function names are descriptive and use verbs. Follow the snake_case convention in Python:

  • ‚úÖ Good Names
  • ‚ùå Bad Names

‚úÖ Good Function Names

def calculate_total_price(quantity, unit_price):
    return quantity * unit_price

def validate_email_address(email):
    return "@" in email and "." in email

def display_student_report(student_name, grades):
    average = sum(grades) / len(grades)
    print(f"{student_name}: {average:.2f}%")

Why these work: Names clearly describe what each function does. Anyone reading the code can understand the purpose without looking at the implementation.

‚ùå Bad Function Names

def calc(q, p):  # What does this calculate?
    return q * p

def check(e):  # Check what?
    return "@" in e and "." in e

def do_stuff(x, y):  # Too vague!
    z = sum(y) / len(y)
    print(f"{x}: {z:.2f}%")

Problems: Abbreviated names (calc), vague verbs (check, do_stuff), and unclear parameters (q, p, e) make code hard to understand and maintain.

üì• Function Parameters

Parameters are variables that receive values when the function is called. They let you pass information into your functions.

def greet(name):  # 'name' is a parameter
    print(f"Hello, {name}!")

greet("Alice")  # "Alice" is the argument
greet("Bob")    # "Bob" is the argument
üí° Parameters vs. Arguments: Parameters are placeholders in the function definition. Arguments are the actual values you pass when calling the function.

Multiple Parameters

Functions can accept multiple parameters:

def calculate_rectangle_area(width, height):
    area = width * height
    print(f"A rectangle {width}√ó{height} has area {area}")

calculate_rectangle_area(5, 10)
calculate_rectangle_area(7, 3)

‚Ü©Ô∏è Return Values

Functions can send data back using the return keyword. This lets you use the result in other parts of your program.

def add_numbers(a, b):
    result = a + b
    return result  # Send the result back

total = add_numbers(5, 3)  # total now equals 8
print(f"The sum is: {total}")

Return vs. Print

Important: return and print() are different. return sends a value back to the caller (useful for calculations). print() only displays text on screen.

Purpose Sends value back to caller Displays text on screen

‚úèÔ∏è Practice: Temperature Converter

Create two functions:

  • celsius_to_fahrenheit(celsius) - converts Celsius to Fahrenheit
  • fahrenheit_to_celsius(fahrenheit) - converts Fahrenheit to Celsius

Formulas: F = (C √ó 9/5) + 32 and C = (F - 32) √ó 5/9

üí° View Solution 
def celsius_to_fahrenheit(celsius):
    return (celsius * 9/5) + 32

def fahrenheit_to_celsius(fahrenheit):
    return (fahrenheit - 32) * 5/9

# Test
print(celsius_to_fahrenheit(25))  # 77.0
print(fahrenheit_to_celsius(77))  # 25.0
